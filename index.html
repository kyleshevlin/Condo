Condo

Because you don't always need a mansion-sized Sass grid library.

Download Latest Release Fork On Github

I'm a big fan of Susy. It's an incredibly robust grid library. I've used it in dozens of projects and even wrote a snippets package for Sublime Text for it.

However, in most of my Susy projects, I only need about 10% of what the library has to offer and the other 90% is wasted. Sitting there. Taking up space.

I wanted to make a grid library that was pared down. Where 100% of it was used, and covered almost all my use cases. In pursuit of that goal, I built Condo.

Like Susy, Condo is a Sass grid library designed to apply grid settings via the presentation layer of your website, i.e. in the CSS, and not in the semantic layer, i.e. your HTML. This is accomplished through a collection of mixins and functions. It differs from Susy in some important ways:

  1. It only handles a limited set of use cases.
  2. It only has the bare essential mixins and functions.
  3. And these mixins and functions are much more strict.

In Susy, many of the mixins and functions allow you to pass a list of values and keywords in as arguments. Condo is much more rigid, each mixin and function expecting a set of parameters with solid defaults, thus eliminating some of the cognitive load of wrapping your head around Susy's powerful capabilities.

The best way to get to know Condo is to read the documentation and use it. You can read the full documentation here. The docs are automatically generated by Sassdoc from comments in the source code. I encourage you to look at the source code itself for a better understanding of how it all functions.
